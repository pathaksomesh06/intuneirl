We will also delve into the advantages and considerations of using the 'number of days' method versus the 'specific date, time, and version' method for tighter update control.</p><hr><h3 id="setting-update-deadlines-based-on-days">Setting Update Deadlines Based on Days</h3><p>Starting with version 4.13 of Microsoft AutoUpdate (MAU), you can set a deadline for when updates are required to be installed on a userâ€™s Mac. </p><p>Users will receive notifications about the upcoming deadline and can temporarily postpone the updates from being installed. But once the deadline is reached, any applications the user has open will be closed and the updates applied.</p><p>Advantages of Setting Update Deadlines Based on Days:</p><ul><li><strong><em>Flexibility in scheduling updates</em>:</strong> By setting update deadlines based on the number of days, you have more flexibility in determining when updates should be enforced. This allows your end users to plan their other activities like meetings, etc.</li><li><strong><em>Prioritization based on urgency</em>:</strong> With the ability to set deadlines based on days, you can also prioritize updates according to their urgency. Critical security updates can be enforced sooner, while non-essential updates can be scheduled for a later date.</li><li><strong><em>Managing update rollouts</em>:</strong> Setting update deadlines based on days is particularly useful when managing update rollouts across different departments or teams. It allows for staggered deployment, ensuring that all systems are updated within a specified timeframe without overwhelming the network or causing disruptions.</li></ul><p>To configure a deadline that is a certain number of days after the update is detected, use the following preference setting:</p><figure class="kg-card kg-image-card"><img src="__GHOST_URL__/content/images/2023/10/image.png" class="kg-image" alt="" loading="lazy" width="1636" height="462" srcset="__GHOST_URL__/content/images/size/w600/2023/10/image.png 600w, __GHOST_URL__/content/images/size/w1000/2023/10/image.png 1000w, __GHOST_URL__/content/images/size/w1600/2023/10/image.png 1600w, __GHOST_URL__/content/images/2023/10/image.png 1636w" sizes="(min-width: 720px) 720px"></figure><p>For example, to configure a specific date and time for a deadline for Word and Outlook you can use something like this:</p><pre><code class="language-xml">&lt;key&gt;UpdateDeadline.ApplicationsForcedUpdateSchedule&lt;/key&gt;

If an application is open and can't be updated, a notification will alert users about an impending deadline. At this point, users have the option to save their work, close the active applications, and allow MAU to proceed with the updates. Doing so will eliminate any further deadline notifications for those specific applications.</p><p>If users prefer not to update immediately, they have the option to delay the updates. In this case, they will receive subsequent reminders as the deadline approaches. Initially, users can opt to be reminded again after a specific number of hours, but postponing beyond the deadline is not an option.</p><p>As the deadline nearsâ€”specifically, one hour before itâ€”users will receive a continuous notification accompanied by a countdown timer. If the deadline is reached and the users have not saved their work or closed their applications, MAU will automatically close the applications, without saving any data, and initiate the updates.</p><p>You have the flexibility to extend the grace period by adjusting the deadline timer settings. The default grace time is 60 minutes, but this can be increased. For instance, if you wish for the countdown to commence 3 hours before the deadline, you can set the timer to 180 minutes.</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="__GHOST_URL__/content/images/2023/10/image-1.png" class="kg-image" alt="" loading="lazy" width="1636" height="628" srcset="__GHOST_URL__/content/images/size/w600/2023/10/image-1.png 600w, __GHOST_URL__/content/images/size/w1000/2023/10/image-1.png 1000w, __GHOST_URL__/content/images/size/w1600/2023/10/image-1.png 1600w, __GHOST_URL__/content/images/2023/10/image-1.png 1636w" sizes="(min-width: 720px) 720px"><figcaption><span style="white-space: pre-wrap;">Deadline preference for MAU</span></figcaption></figure><p>The simplest setting to configure is the "<strong>number of days</strong>" deadline for all applications after an update is identified. This setting can be adjusted at the same top-tier level of the configuration profile, within the 'mcx_preference_settings' key dictionary, alongside other parameters like 'HowToCheck,' 'UpdateCheckFrequency,' and 'ChannelName.' Even with this global setting in place, you still have the flexibility to specify different deadlines for individual applications further down in the profile.</p><div class="kg-card kg-callout-card kg-callout-card-blue"><div class="kg-callout-emoji">ðŸ’¡</div><div class="kg-callout-text">Ensure that you configure the IgnoreUIOpenAfterInstall setting in Microsoft AutoUpdate (MAU) as it is used to control the behavior of applications after they have been updated.
